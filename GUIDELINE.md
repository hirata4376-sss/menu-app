# 🍳 プロジェクト指針：献立アプリ（残り物レシピ管理）

## 1. 目的・ターゲット
- **目的**: 冷蔵庫の食材を使い切る。1週間分の献立を考える手間を削減する。
- **ターゲット**: 自分と妻（夫婦）。

## 2. 利用シーン
- 休日のまとめ買い前に、スマホのブラウザからサクッと操作できること。

## 3. MVP（最小限の実用構成）の範囲
- **献立の登録**: メニュー名と使用する食材（タグ形式）をセットで保存できる。
- **献立の一覧・編集・削除**: 登録した献立を一覧表示し、編集・削除ができる。
- **食材逆引き検索**: 「にんじん」「豚肉」など食材名を入力すると、それを使う献立を絞り込み表示できる。
- **データ保存**: ローカルストレージを使用（ログイン不要で即使える）。
- **UI方針**: モバイルファースト。スマホで片手操作しやすいレイアウト。

## 4. 技術スタック
おみくじアプリ（Next.js）の経験を活かし、同じ構成を採用します。
- **フレームワーク**: Next.js (React)
- **スタイリング**: Vanilla CSS (またはCSS Modules)
- **デプロイ環境**: Vercel（おみくじアプリと同様）
